# 介绍

`antdv-superform` 是一个基于 `Vue 3` 和 `Ant Design Vue 3` 的声明式业务组件库，核心目标是用一份配置描述业务结构，并将其渲染为表单、详情、表格、按钮区或弹窗内容。

它不是对 Ant Design Vue 的简单二次封装，而是更偏“业务层编排”的组件方案。你提供 schema，它负责：

- 组织页面结构
- 绑定字段值
- 执行校验
- 处理联动逻辑
- 在不同展示形态之间复用配置

## 解决的问题

传统中后台页面通常会遇到这些重复劳动：

- 表单页和详情页字段几乎一致，却要维护两套视图
- 列表页搜索项和编辑弹窗字段重复定义
- 相同的按钮权限、默认样式、交互行为散落在各页面
- 动态显隐、禁用、依赖计算逻辑难以统一收敛

`antdv-superform` 的思路是把这些共性能力沉到 schema 和 hooks 中，尽量减少模板层重复代码。

## 核心能力

- `SuperForm`：配置化表单
- `SuperTable`：带搜索、分页、编辑和详情能力的业务表格
- `SuperDetail`：详情展示视图
- `SuperButtons`：按钮组渲染
- `useModal` / `useModalForm`：业务弹窗能力
- `defineForm` / `defineTable` / `defineDetail`：为 schema 提供类型收敛

## 适用场景

- 管理后台 CRUD 页面
- 数据配置页面
- 查询 + 表格 + 编辑弹窗的一体化页面
- 希望通过配置驱动 UI 的中后台项目

## 不适合的场景

- 高度自由的营销页或品牌官网
- 强交互、强视觉定制且几乎没有重复结构的前台页面
- 不希望通过 schema 维护页面结构的项目
